# Groq AI Provider Setup

The AI layer uses the **Groq API** to produce structured JSON semantic plans.
It **never calculates metrics directly** — all grouping, aggregation, sorting, filtering, and
`chartData` generation is performed exclusively by backend Pandas code.

## How it works

1. User uploads a dataset.
2. The dataset preview is shown immediately (no waiting for AI).
3. `/api/ai/fast-dashboard/{dataset_id}` is called in the background.
4. Groq identifies semantic columns only (domain, entity, metrics, dimensions, time columns).
5. Pandas builds all KPI values and chart data from the semantic plan.
6. If Groq times out or fails, the dataset preview remains visible with a clear message.

## Setup

### 1. Create a Groq API Key

Go to [console.groq.com](https://console.groq.com) → **API Keys** → Create a new key.

### 2. Add GROQ_API_KEY to backend/.env

```text
AI_PROVIDER=groq
GROQ_API_KEY=your_api_key_here
GROQ_MODEL=llama-3.1-8b-instant
AI_SEMANTIC_TIMEOUT_SECONDS=8
AI_CHAT_TIMEOUT_SECONDS=8
AI_MAX_COLUMN_CARDS=25
AI_MAX_RECOMMENDED_CHARTS=5
```

### 3. Install requirements

```bash
cd backend
pip install -r requirements.txt
```

### 4. Start the backend

```bash
uvicorn main:app --reload --port 8000
```

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `AI_PROVIDER` | `groq` | AI provider selection |
| `GROQ_API_KEY` | — | Your Groq API key (required) |
| `GROQ_MODEL` | `llama-3.1-8b-instant` | Groq model to use |
| `AI_SEMANTIC_TIMEOUT_SECONDS` | `8` | Timeout for semantic planning |
| `AI_CHAT_TIMEOUT_SECONDS` | `8` | Timeout for chat/prompt charts |
| `AI_MAX_COLUMN_CARDS` | `25` | Max columns sent to Groq |
| `AI_MAX_RECOMMENDED_CHARTS` | `5` | Max charts to generate |

## AI Principle

> The LLM provider **only identifies** semantic column roles:
> domain, primary entity, important metrics, dimensions, and time columns.
>
> All KPI values, groupby aggregations, sorting, filtering, and `chartData`
> are **always generated by Pandas** — never by the LLM.

## Missing or invalid GROQ_API_KEY

If `GROQ_API_KEY` is not set or is the placeholder value:
- The app will **not crash**.
- `/api/ai/fast-dashboard` will return `status="error"` with a safe message.
- The dataset preview will remain visible.

## Manual validation tests

1. **Football dataset** — columns: `Player`, `Team`, `Position`, `Goals`, `Assists`, `Minutes`, `Rating`, `xG`
   - Expected: `detected_domain` = `football_soccer`
   - `primary_entity` = `Player`
   - `primary_metrics` includes `Goals`, `Assists`, `Rating` or `xG`
   - Charts include top goals by player and goals vs assists scatter

2. **Sales dataset** — columns: `Product`, `Region`, `Sales`, `Profit`, `Quantity`, `Order_Date`
   - Expected: `primary_metrics` = `[Sales, Profit, Quantity]`
   - `dimensions` includes `Product`, `Region`
   - `time_columns` = `[Order_Date]`

3. **Chat prompt**: `gol ve asistleri oyuncuya göre göster`
   - Expected: `mode="groq_prompt_chart"` when Groq responds within timeout.
   - Should not randomly select Age or unrelated columns.

4. **Missing API key**:
   - Expected: `/api/ai/fast-dashboard` returns `{"status": "error", ...}` safely.
   - App continues to function normally with dataset preview.
