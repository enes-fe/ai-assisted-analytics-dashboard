# AI Provider Setup

The primary dashboard AI path uses the Groq API to produce structured JSON semantic plans. The LLM identifies dataset roles and intended analysis only. All grouping, aggregation, sorting, filtering, KPI values, forecasts, clusters, and `chartData` are generated by backend Pandas/scikit-learn code.

## How It Works

1. The user uploads one or more tabular files.
2. The dataset preview is shown immediately.
3. `/api/ai/fast-dashboard/{dataset_id}` runs the fast semantic planner.
4. Groq identifies semantic roles such as domain, entity, metrics, dimensions, and time columns.
5. Backend services validate the plan and build deterministic KPI/chart data.
6. If the AI call fails or times out, the app keeps the preview visible and returns a safe message.

## Setup

Copy the committed example file:

```powershell
Copy-Item backend\.env.example backend\.env
```

Then edit `backend/.env` and set a real key:

```text
AI_PROVIDER=groq
GROQ_API_KEY=your_real_key_here
GROQ_MODEL=llama-3.1-8b-instant
AI_SEMANTIC_TIMEOUT_SECONDS=8
AI_CHAT_TIMEOUT_SECONDS=8
AI_CLUSTER_NAMING_ENABLED=true
AI_CLUSTER_NAMING_TIMEOUT_SECONDS=4
AI_MAX_COLUMN_CARDS=25
AI_MAX_RECOMMENDED_CHARTS=5
```

Install backend requirements:

```powershell
pip install -r backend\requirements.txt
```

Start the backend from the `backend` directory:

```powershell
cd backend
uvicorn main:app --reload --port 8000
```

## Environment Variables

| Variable | Default | Purpose |
|---|---:|---|
| `AI_PROVIDER` | `groq` | Provider label for the primary AI path |
| `GROQ_API_KEY` | placeholder | Groq API key for semantic planning |
| `GROQ_MODEL` | `llama-3.1-8b-instant` | Groq model name |
| `AI_SEMANTIC_ENABLED` | `true` | Enables semantic-plan debug endpoint |
| `AI_SEMANTIC_TIMEOUT_SECONDS` | `8` or `20` | Timeout for semantic planning |
| `AI_CHAT_ENABLED` | `true` | Enables prompt-to-chart chat planning |
| `AI_CHAT_TIMEOUT_SECONDS` | `8` or `15` | Timeout for prompt chart planning |
| `AI_CLUSTER_NAMING_ENABLED` | `true` | Enables optional AI fallback for generic cluster names |
| `AI_CLUSTER_NAMING_TIMEOUT_SECONDS` | `4` | Timeout for optional cluster naming |
| `AI_MAX_COLUMN_CARDS` | `25` | Maximum profiled columns sent to the planner |
| `AI_MAX_RECOMMENDED_CHARTS` | `5` | Maximum AI-recommended charts |
| `OLLAMA_HOST` | `http://localhost:11434` | Legacy/local semantic-planner host |
| `OLLAMA_MODEL` | `qwen2.5-coder:3b` | Legacy/local semantic-planner model |
| `OLLAMA_NUM_CTX` | `2048` | Legacy/local context window |

## AI Principle

The LLM provider may select semantic roles and chart intent. It must not calculate results directly. Backend code remains the source of truth for:

- KPI values
- groupby aggregations
- sorting and filtering
- chart series and labels
- forecasting
- clustering
- advanced statistical outputs

## Missing Or Invalid Key

If `GROQ_API_KEY` is missing or still set to the placeholder value:

- The app should not crash.
- `/api/ai/fast-dashboard/{dataset_id}` returns a safe error response.
- Dataset preview and non-AI flows remain available.

## Manual Validation

1. Football-style dataset: columns such as `Player`, `Team`, `Position`, `Goals`, `Assists`, `Minutes`, `Rating`, and `xG`.
   - Expected domain: football/soccer.
   - Expected entity: player.
   - Expected metrics: goals, assists, rating, or xG.
   - Expected charts: top goals by player and a goals-vs-assists style chart.

2. Sales-style dataset: columns such as `Product`, `Region`, `Sales`, `Profit`, `Quantity`, and `Order_Date`.
   - Expected metrics: sales, profit, and quantity.
   - Expected dimensions: product and region.
   - Expected time column: order date.

3. Chat prompt: `show goals and assists by player`.
   - Expected mode: `groq_prompt_chart` when Groq responds within timeout.
   - The planner should not select unrelated columns.

4. Missing API key.
   - Expected: the AI endpoint returns a safe error.
   - The upload preview remains usable.
