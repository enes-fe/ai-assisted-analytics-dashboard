# AI-Assisted Automatic Analytics Dashboard

Final thesis project for an AI-assisted analytics dashboard. The app lets a user upload tabular data, inspect the dataset, and generate a focused dashboard with KPI cards, charts, forecasting, clustering, and optional/debug statistical outputs.

## Architecture

- `src/` contains the Vite/React frontend.
- `backend/main.py` exposes the FastAPI API.
- `backend/services/` contains deterministic analytics logic for KPIs, charts, forecasting, clustering, statistics, and shared utilities.
- `backend/services/ai/` contains the AI semantic planning layer.
- `scratch/` contains lightweight project smoke/audit scripts.

The AI layer is intentionally narrow: Groq/LLM calls identify semantic column roles and analysis intent. All real calculations, aggregations, KPI values, chart data, forecasting, clustering, and statistical outputs are generated by deterministic backend code.

## Backend

Run the API from the backend directory so local imports resolve as expected:

```powershell
cd backend
uvicorn main:app --reload
```

Important local configuration lives in `backend/.env`. Do not commit API keys, local datasets, database files, uploads, generated exports, virtual environments, or build artifacts.

## Frontend

```powershell
npm install
npm run dev
```

Useful checks:

```powershell
npm run lint
npm run build
```

## Validation

Python syntax validation:

```powershell
python -m compileall backend/services backend/main.py
```

Existing project smoke/audit scripts are kept under `scratch/` and can be run when the local Python environment is available.

## Thesis Scope Notes

- The default dashboard scope should remain narrow and defensible.
- Advanced statistical tests are opt-in/debug behavior, not the default dashboard claim.
- Existing endpoint names, response fields, and frontend behavior should remain backward compatible.
